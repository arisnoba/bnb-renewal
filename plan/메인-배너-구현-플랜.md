# 메인 배너 구현 플랜

> 기준: `plan/todo.md`의 `## 메인` 항목과 2026-05-27 인터뷰 답변.
> 목표: Payload 관리자에서 센터별 메인 배너를 등록하고, 실제 센터 페이지 첫 화면에서 MVP로 검증한다.

## 확정 요구사항

- 글로벌 그룹/글로벌 컬렉션의 표시 이름은 `메인`으로 변경한다.
- `메인`은 Payload 관리자 글로벌 컬렉션 그룹 안에 위치한다.
- 홈(`/`)에는 메인 배너를 노출하지 않는다.
  - 홈은 게이트 페이지 성격으로 유지한다.
- 센터별 실제 페이지에서 배너를 테스트하고 노출한다.
  - 아트센터: `/art`
  - 입시센터: `/exam`
  - 키즈센터: `/kids`
  - 하이틴센터: `/highteen`
  - 애비뉴센터: `/avenue`
- 배너는 센터별로 여러 개 등록 가능해야 한다.
- 프론트 노출은 Swiper 방식이다.
- 자동 롤링은 사용하되, 시간/세부 옵션은 나중에 조정 가능하게 둔다.
- 배너에는 공개 상태와 예약 노출 기간이 필요하다.
- 공통 배너는 만들지 않는다. 각 배너는 반드시 하나의 센터에 소속된다.
- 영상과 이미지가 함께 있으면 영상이 우선 노출된다.
- 이미지는 영상의 poster 역할을 한다.
- 배너 텍스트는 실제 프론트 화면에 노출한다.
- 배너 클릭 동작은 없다.
- 연결 콘텐츠는 배너 하단 또는 오버레이 영역에서 marquee로 노출하며, 각 marquee 아이템이 클릭 대상이다.

## 관리자 모델

### 글로벌

기존 글로벌 그룹 또는 글로벌 컬렉션의 관리자 표시 이름을 `메인`으로 바꾼다.

후보 구조:

```txt
메인
- 메인 배너
- SNS
```

`SNS`는 같은 `메인` 글로벌 안의 하단 섹션으로 둔다. 단, 이번 MVP 우선순위는 메인 배너다.

### 배너 배열

`메인` 글로벌 안에 `mainBanners` 배열을 둔다.

필드 초안:

| 필드명 | 타입 | 필수 | 설명 |
|---|---:|---:|---|
| `title` | text | 예 | 실제 배너 제목 |
| `broadcaster` | text | 아니오 | 방송사/채널명 |
| `description` | textarea | 아니오 | 실제 배너 설명 |
| `center` | select | 예 | `art`, `exam`, `kids`, `highteen`, `avenue` 중 하나 |
| `status` | select | 예 | `draft`, `published` |
| `publishStartAt` | date | 아니오 | 예약 노출 시작일 |
| `publishEndAt` | date | 아니오 | 예약 노출 종료일 |
| `desktopImage` | upload(media) | 조건부 | 웹 이미지/poster |
| `mobileImage` | upload(media) | 아니오 | 모바일 이미지/poster |
| `desktopVideo` | upload(media) | 아니오 | 웹 영상 |
| `mobileVideo` | upload(media) | 아니오 | 모바일 영상 |
| `linkedProfiles` | relationship | 아니오 | 연결 배우/프로필 다수 |
| `linkedExamReviews` | relationship | 아니오 | 입시센터 연결 합격후기 다수 |

### 센터 선택 규칙

`center`는 필수 단일 선택이다.

| 라벨 | 값 |
|---|---|
| 아트센터 | `art` |
| 입시센터 | `exam` |
| 키즈센터 | `kids` |
| 하이틴센터 | `highteen` |
| 애비뉴센터 | `avenue` |

`공통` 값은 만들지 않는다.

### 미디어 필수 규칙

관리자 UX와 프론트 안정성을 위해 최소 poster 역할을 하는 이미지는 요구한다.

권장 검증:

- `desktopImage`는 필수.
- `mobileImage`는 선택. 없으면 `desktopImage`를 사용한다.
- `desktopVideo`는 선택.
- `mobileVideo`는 선택. 없으면 `desktopVideo`를 사용한다.
- 영상이 있으면 영상 우선, 이미지는 poster로 사용한다.
- 영상이 없어도 이미지 배너로 노출 가능하다.

운영자가 모바일 전용 이미지를 자주 누락할 수 있으므로, MVP에서는 `mobileImage`를 선택으로 두고 fallback을 제공한다.

### 연결 콘텐츠 규칙

모든 센터는 여러 개의 연결 콘텐츠를 가질 수 있다.

센터별 연결 후보:

| 센터 | 연결 콘텐츠 |
|---|---|
| 아트센터 | 배우/프로필 다수 |
| 키즈센터 | 배우/프로필 다수 |
| 하이틴센터 | 배우/프로필 다수 |
| 입시센터 | 합격후기 다수 |
| 애비뉴센터 | MVP에서 확인 필요 |

입시센터는 기존 요구사항 기준으로 `합격후기`를 연결한다.

애비뉴센터는 배너 대상에는 포함하지만, 연결 콘텐츠의 실제 컬렉션은 구현 전 확인이 필요하다. MVP에서는 연결 콘텐츠 없이 배너만 노출하거나, 연결 가능한 운영 컬렉션이 확인되면 그 컬렉션을 붙인다.

### 관리자 검증

- `title` 누락 시 필드 단위 validation 메시지를 보여준다.
- `center` 누락 시 필드 단위 validation 메시지를 보여준다.
- `status` 누락 시 필드 단위 validation 메시지를 보여준다.
- `desktopImage` 누락 시 필드 단위 validation 메시지를 보여준다.
- `publishEndAt`이 `publishStartAt`보다 빠르면 종료일 필드에 validation 메시지를 보여준다.
- Payload 컬렉션/글로벌 검증은 `beforeValidate` throw보다 field-level `validate`를 우선한다.

## 프론트 노출 규칙

### 대상 페이지

실제 센터 URL에서 MVP를 검증한다.

| URL | 센터 |
|---|---|
| `/art` | 아트센터 |
| `/exam` | 입시센터 |
| `/kids` | 키즈센터 |
| `/highteen` | 하이틴센터 |
| `/avenue` | 애비뉴센터 |

홈(`/`)에는 노출하지 않는다.

### 배너 필터링

페이지 센터와 같은 `center` 값을 가진 배너만 노출한다.

노출 조건:

- `status`가 `published`
- `publishStartAt`이 없거나 현재 시각 이후가 아님
- `publishEndAt`이 없거나 현재 시각 이전이 아님
- 해당 센터와 일치

정렬:

- 배열 순서를 기본 노출 순서로 사용한다.
- 별도 `sortOrder` 필드는 MVP에서는 두지 않는다.

### Swiper 동작

MVP 규칙:

- 배너가 2개 이상이면 Swiper 활성화.
- 배너가 1개면 정지 화면으로 노출.
- 자동 롤링은 활성화하되, 옵션은 한 곳에서 조정 가능하게 둔다.
- 나중에 롤링 속도, pagination, navigation, fade/slide 효과를 수정할 수 있게 컴포넌트 옵션을 분리한다.

### 미디어 렌더링

우선순위:

1. 모바일 viewport + `mobileVideo` 있음: `mobileVideo`
2. 모바일 viewport + `desktopVideo` 있음: `desktopVideo`
3. 데스크톱 viewport + `desktopVideo` 있음: `desktopVideo`
4. 모바일 viewport + `mobileImage` 있음: `mobileImage`
5. fallback: `desktopImage`

영상 속성:

- autoplay
- muted
- loop
- playsInline
- controls 없음
- poster는 viewport에 맞는 이미지 사용

이미지/영상은 첫 화면 히어로 배경으로 사용하므로, 어둡고 흐린 장식 이미지가 아니라 실제 인물/작품 상태가 보이는 미디어를 기준으로 한다.

### 텍스트 렌더링

배너 위에 실제 텍스트를 노출한다.

노출 필드:

- `title`
- `broadcaster`
- `description`

기본 배치:

- 데스크톱: 좌측 하단 또는 좌측 중앙
- 모바일: 하단 영역, 텍스트가 미디어 주요 피사체를 가리지 않게 배치

### Marquee 연결 콘텐츠

배너 자체는 클릭하지 않는다.

배너에 연결된 여러 콘텐츠는 marquee 영역에서 노출한다.

동작:

- marquee 아이템 각각이 링크 대상이다.
- 아트/키즈/하이틴은 연결된 배우/프로필 상세로 이동한다.
- 입시는 연결된 합격후기 상세로 이동한다.
- 연결 콘텐츠가 없으면 marquee 영역은 숨긴다.

## 구현 순서

### 1단계: 현재 구조 확인

- 현재 글로벌 컬렉션 목록 확인.
- `footer` 또는 유사 글로벌의 관리자 그룹 표시 방식 확인.
- 실제 센터 URL이 `pages` 기반인지, 정적 라우트인지 확인.
- 센터별 프론트 렌더링 진입점을 확인.
- 배우/프로필 컬렉션과 합격후기 컬렉션의 slug/detail URL 규칙 확인.

### 2단계: Payload 모델 설계

- `메인` 글로벌 표시명 변경.
- `mainBanners` 배열 추가.
- 센터 select 추가.
- 공개 상태/예약 노출 필드 추가.
- 미디어 upload 필드 추가.
- 연결 relationship 필드 추가.
- row label은 `센터 + 제목`으로 구성.

### 3단계: 마이그레이션 및 타입 생성

컬렉션/글로벌 변경 후 필수 절차:

```bash
npm run db:local:migrate:create
npm run db:local:migrate
npm run payload:generate-types
npm run payload:generate-importmap
```

글로벌 구조 변경이 DB에 어떤 diff를 만드는지 먼저 확인하고, 불필요한 schema churn은 피한다.

### 4단계: 프론트 MVP

- 센터 페이지에서 해당 센터의 배너만 조회한다.
- `MainBannerSwiper` 컴포넌트 추가.
- 영상/이미지 fallback 로직 구현.
- 텍스트 오버레이 구현.
- 연결 콘텐츠 marquee 구현.
- `/art`, `/exam`, `/kids`, `/highteen`, `/avenue`에서 실제 노출 확인.

### 5단계: 검증

필수 검증:

```bash
npm run typecheck
npm run lint
```

수동 검증:

- 각 센터 페이지에서 해당 센터 배너만 보이는지 확인.
- 공개/비공개 상태가 반영되는지 확인.
- 예약 시작/종료일이 반영되는지 확인.
- 영상이 있으면 영상이 우선 노출되는지 확인.
- 영상이 없으면 이미지가 노출되는지 확인.
- 모바일 미디어 fallback이 동작하는지 확인.
- 연결 콘텐츠 marquee가 개별 링크로 이동하는지 확인.

가능하면 브라우저 캡처:

- 데스크톱 `/art`
- 모바일 `/art`
- 데스크톱 `/exam`
- 모바일 `/exam`

## 미확정 항목

- 애비뉴센터에서 marquee로 연결할 실제 콘텐츠 컬렉션.
- 배우/프로필 상세 URL의 최종 public route.
- 합격후기 상세 URL의 최종 public route.
- Swiper 자동 롤링 시간.
- 텍스트 오버레이의 최종 디자인 기준.

## MVP 완료 기준

- Payload 관리자에서 `메인` 글로벌 안에 센터별 배너를 등록할 수 있다.
- 배너별 공개 상태와 예약 노출을 설정할 수 있다.
- 센터별 실제 URL에서 해당 센터 배너만 노출된다.
- 배너가 여러 개이면 Swiper로 동작한다.
- 영상이 있으면 영상 우선, 없으면 이미지로 노출된다.
- 배너 텍스트가 프론트에 표시된다.
- 연결 콘텐츠가 있으면 marquee 아이템으로 노출되고, 아이템 단위로 이동한다.
- `npm run typecheck`, `npm run lint`가 통과한다.

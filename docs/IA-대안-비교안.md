# 배우앤배움 사이트 IA 대안 비교안

## 1. 목적

프로젝트 개요가 `배우앤배움 통합 사이트 + 5개 센터 하위 운영` 구조로 바뀌면서, 메뉴 설계 방향도 함께 재검토가 필요해졌습니다.

이번 문서는 아래 2가지 안을 비교하기 위한 자료입니다.

1. 통합안
2. 센터중심안

핵심 쟁점은 다음입니다.

- 공통 콘텐츠를 얼마나 통합할 것인가
- 센터 고유성을 얼마나 전면에 드러낼 것인가
- 사용자가 `브랜드/센터 중심`으로 탐색하는지, `목적/콘텐츠 중심`으로 탐색하는지

---

## 2. 대안 1. 통합안

### 개념

센터는 필터 조건으로 처리하고, 사이트의 1차 메뉴는 공통 콘텐츠 카테고리 중심으로 구성하는 방식입니다.

### 예시 구조

```text
1차 메뉴
- 게이트
- 배우앤배움 소개
- 센터 소개
- 프로그램
- 강사진
- 성과/활동
- 캐스팅/오디션
- 이용안내
- 상담/지원
```

### 특징

- 공통 메뉴를 하나로 합쳐서 운영 효율이 높음
- 중복 페이지를 줄이기 쉬움
- 필터와 태그 중심 구조에 적합
- 내부 CMS/데이터 모델을 통합하기 쉬움

### 장점

- 운영/개발 효율이 가장 좋음
- 콘텐츠 확장성과 검색 구조 정리에 유리함
- 공통 템플릿과 공통 컴포넌트 사용이 쉬움
- 장기적으로 브랜드 허브형 플랫폼 구조에 적합

### 단점

- 각 센터의 브랜드 개성이 약하게 보일 수 있음
- 사용자가 특정 센터를 바로 찾을 때 한 단계 더 생각해야 함
- 기존 사용자에게는 “센터가 사라졌다”는 인상을 줄 수 있음
- 입시센터, 애비뉴센터처럼 성격이 강한 센터는 희석될 위험이 있음

### 적합한 경우

- 브랜드 통합을 강하게 밀고 싶을 때
- 운영팀이 공통 구조로 관리 효율을 최우선할 때
- 장기적으로 콘텐츠 플랫폼형 사이트로 키우고 싶을 때

---

## 3. 대안 2. 센터중심안

### 개념

상단 1차 메뉴는 센터 중심으로 노출하되, 실제 내부 정보 구조와 템플릿은 통합형으로 설계하는 방식입니다.

즉,

- 겉으로 보이는 메뉴는 `센터 중심`
- 실제 운영 구조는 `통합 카테고리 중심`

으로 가져갑니다.

### 예시 구조

```text
1차 메뉴
- 배우앤배움
- 아트센터
- 입시센터
- 하이틴센터
- 키즈센터
- 애비뉴센터
- 오디션
- 상담하기
```

각 센터 내부 공통 2차 메뉴:

```text
- 센터소개
- 프로그램
- 강사진
- 성과/활동
- 캐스팅/오디션
- 이용안내
- 상담
```

### 특징

- 1차 인지는 센터 브랜드로 가져감
- 2차 운영은 공통 구조로 맞춤
- 센터별 고유 페이지는 유지하되, 템플릿과 데이터는 통합

### 장점

- 센터 고유성과 운영 효율을 동시에 잡기 좋음
- 사용자는 센터 기반으로 바로 진입할 수 있음
- 내부적으로는 프로그램/강사진/성과를 통합 모델로 관리할 수 있음
- SEO와 운영 확장성도 확보 가능

### 단점

- 설계 난이도가 가장 높음
- 메뉴/템플릿/필터 체계를 처음부터 명확히 잡아야 함
- 겉보기에는 센터별 사이트 같지만 내부는 통합형이라 기획 기준이 흔들리면 구조가 꼬일 수 있음

### 적합한 경우

- 센터 개성도 중요하고 통합 운영도 포기할 수 없을 때
- 브랜드 경험과 운영 효율을 동시에 챙겨야 할 때
- 이번 프로젝트가 “완전 통합”도 아니고 “완전 분리”도 아닌 중간 모델일 때

## 4. 2안 비교표

| 항목                 | 통합안    | 센터중심안 |
| -------------------- | --------- | ---------- |
| 센터 브랜드 노출     | 낮음      | 높음       |
| 운영 효율            | 매우 높음 | 높음       |
| 중복 콘텐츠 관리     | 쉬움      | 보통~쉬움  |
| 사용자 직관성        | 보통      | 높음       |
| 개발 구조 통합       | 매우 쉬움 | 보통~쉬움  |
| SEO/콘텐츠 통합 운영 | 매우 유리 | 유리       |
| 센터 개성 유지       | 약함      | 강함       |
| 기획 난이도          | 보통      | 높음       |
| 장기 확장성          | 높음      | 높음       |

## 5. 실무 적용 권장안

### 권장 1차 메뉴

```text
- 배우앤배움
- 아트센터
- 입시센터
- 하이틴센터
- 키즈센터
- 애비뉴센터
- 오디션
- 상담하기
```

### 배우앤배움 메뉴

- 브랜드 소개
- 교육 철학 / 시스템
- 전체 센터 보기
- 연혁
- 시설 / 위치

### 각 센터 공통 2차 메뉴

- 센터소개
- 프로그램
- 강사진
- 성과/활동
- 캐스팅/오디션
- 이용안내
- 상담

### 오디션 메뉴

- 전체 오디션 안내
- 센터별 오디션
- 지원하기
- 진행 현황

### 상담하기 메뉴

- 통합 상담 신청
- 센터별 상담 안내
- FAQ
- 오시는 길

---

## 6. 추가 의견

현재 사용자 경험 기준으로 보면, `오디션`은 별도 1차 메뉴로 둘 수 있습니다. 실제 전환성과 관심도가 높기 때문입니다.

반면 `상담센터`는 이름 그대로 메뉴화하기보다 `상담하기` 또는 `무료상담`처럼 CTA형 메뉴로 두는 편이 더 좋습니다.

---

## 7. 결론

두 가지 안 중 추천 우선순위는 아래와 같습니다.

1. 센터중심안
2. 통합안

정리하면,

- `통합안`은 운영 효율은 좋지만 현재 BNB 구조에는 다소 과감함
- `센터중심안`은 센터의 개성을 살리면서도 통합 운영 구조를 만들 수 있어 가장 현실적임

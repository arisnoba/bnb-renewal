# News Migration Retention Policy

작성일: 2026-05-13

## 목적

`news`는 총 6,255건으로 데이터와 이미지 수가 많다. 아직 어떤 글을 최종 공개/보존할지 요건이 확정되지 않았으므로, 이미지 업로드와 media relation 연결은 전체 대상에 바로 실행하지 않는다.

이 문서는 2차 정제 단계의 기준을 남긴다. 목표는 불필요한 `media` row와 스토리지 사용을 막기 위해, 3차 이미지 처리 전에 명시적인 대상 목록을 확정하는 것이다.

## 현재 원칙

- 뉴스 글 삭제는 이번 단계에서 하지 않는다.
- `media` 컬렉션에 대량 이미지 업로드를 하지 않는다.
- `thumbnailMedia`, `meta.image`, 본문 이미지 처리는 명시적인 allowlist에 포함된 news만 대상으로 한다.
- 2020년 이전 뉴스는 즉시 삭제하거나 숨기지 않고, 우선 검토 큐로 분리한다.
- 최종 보존/제외 기준이 확정되기 전에는 `bodyHtml`, `thumbnailPath`, `legacyMeta` fallback을 제거하지 않는다.

## 2차 확정 기준

### 콘텐츠 보존

현재는 전체 6,255건을 DB에 보존한다.

최종 공개/삭제/숨김 정책은 별도 결정이 필요하다. 이 단계에서는 실제 삭제 또는 공개 상태 변경을 하지 않는다.

### 이미지 처리 대상

3차 media 처리 확정 allowlist는 `published_at >= 2020-01-01`인 news로 제한한다.

이 기준은 삭제 기준이 아니라 이미지 업로드/연결 범위 제한 기준이다. 2020년 이전 또는 보정 날짜 데이터는 검토 큐로 남기며, 이번 media 처리 범위에서는 제외한다.

# 배우앤배움 관리자 운영 실무자 인터뷰 자료

> 이 문서는 두 파트로 구성됩니다.
>
> - **Part A**: 실무자에게 사전 배포하는 설문 (12문항, 약 10분)
> - **Part B**: 인터뷰어가 구두로 진행하는 후속 질문 (7개)
>
> 데이터 이전/마이그레이션 범위는 기술 검토 단계에서 별도 확인합니다.

---

# Part A. 사전 설문지 (실무자 배포용)

## 섹션 1. 기본 정보

**Q1. 소속 센터를 선택해주세요.**

- 타입: 단일선택
- 아트센터 / 입시센터 / 하이틴센터 / 키즈센터 / 애비뉴센터 / 본사 공통 운영 / 기타

---

**Q2. 현재 담당 역할을 모두 선택해주세요.**

- 타입: 복수선택
- 공지/게시글 등록
- 프로그램 정보 관리
- 강사진 정보 관리
- 후기/성과 등록
- 캐스팅/오디션 관리
- 상담 신청 확인 및 처리
- 운영 승인/검수
- 기타

---

**Q3. 관리자 페이지를 주로 사용하는 기기는 무엇인가요?**

- 타입: 복수선택
- 데스크톱 PC / 노트북 / 태블릿 / 모바일

---

## 섹션 2. 현재 운영 업무

**Q4. 현재 가장 자주 등록하거나 수정하는 콘텐츠를 1~3개 선택해주세요.**

- 타입: 복수선택 (최대 3개)
- 공지사항
- 프로그램/과정 소개
- 강사진 소개
- 합격/출연/성과
- 후기
- 캐스팅/오디션 공고
- 상담 신청 확인
- 배너/메인 비주얼
- FAQ
- 기타

---

**Q5. 콘텐츠를 등록하거나 수정할 때 현재 어떤 순서로 진행하나요?**

- 타입: 단답형 또는 장문형
- 예시: "초안 작성 → 팀장 확인 → 사진 첨부 → 게시" / "카카오톡으로 내용 받아서 직접 등록"

---

## 섹션 3. 센터별 운영 범위

**Q6. 아래 항목 중 센터별로 따로 관리해야 하는 것을 모두 선택해주세요.**

- 타입: 복수선택
- 프로그램/과정
- 강사진
- 성과/후기
- 캐스팅/오디션
- 공지사항
- 상담 신청
- FAQ
- 배너/메인 비주얼
- 기타

---

**Q7. 한 번 등록 후 여러 센터에 공통으로 노출해도 된다고 생각하는 항목을 모두 선택해주세요.**

- 타입: 복수선택
- 배우앤배움 브랜드 소개
- 공통 시설안내
- 공통 정책 페이지 (이용약관, 환불정책 등)
- 공통 FAQ
- 공통 공지
- 공통 강사진
- 공통 상담안내
- 기타

---

**Q8. 같은 콘텐츠를 여러 센터에 동시에 노출해야 하는 경우가 있나요?**

- 타입: 단일선택
- 매우 자주 있다 / 가끔 있다 / 거의 없다 / 전혀 없다

---

## 섹션 4. 현재 관리자 페이지 문제점

**Q9. 현재 관리자 페이지에서 자주 불편한 문제를 모두 선택해주세요.**

- 타입: 복수선택
- 메뉴 구조가 복잡하다
- 콘텐츠 찾기가 어렵다
- 같은 내용을 여러 번 입력해야 한다
- 이미지/파일 업로드가 불편하다
- 검색/필터가 약하다
- 권한 구분이 불명확하다
- 신청자 관리가 어렵다
- 모바일 사용성이 낮다
- 게시 상태 관리가 어렵다
- 기타

---

**Q10. 가장 자주 불편한 점을 한 가지만 직접 적어주세요.**

- 타입: 단답형

---

## 섹션 5. 새 관리자 페이지 요구사항

**Q11. 새 관리자 페이지에서 꼭 유지되었으면 하는 기능을 적어주세요.**

- 타입: 단답형 또는 장문형
- 예시: "공지 상단 고정" / "센터별 게시판 구분" / "지원서 다운로드"

---

**Q12. 새 관리자 페이지에서 가장 개선되었으면 하는 기능을 최대 3개 선택해주세요.**

- 타입: 복수선택 (최대 3개)
- 콘텐츠 한 번 등록 후 여러 센터 동시 노출
- 더 쉬운 이미지 업로드
- 검색/필터 강화
- 예약 발행
- 권한 분리
- 신청자 상태 관리 (신규/확인중/완료 등)
- 복사 등록 기능
- 모바일 사용성 개선
- 승인 워크플로
- 기타

# Part B. 후속 인터뷰 가이드 (인터뷰어용)

> 설문 응답을 검토한 뒤 구두로 진행합니다.
> 답변을 유도하기보다 "실제 있었던 사례"를 이끌어내는 것이 목표입니다.

---

**IQ1. 최근 한 달 동안 관리자 페이지에서 실제로 처리한 업무를 구체적으로 말씀해주세요.**

- 의도: 추상적인 요구가 아닌 실제 업무 사례 확보
- 예: "입시센터 합격생 소식 3건 등록, 상담 신청자 확인, FAQ 문구 수정"

---

**IQ2. 상담 신청이 들어오면 실제로 누가 확인하고 어떻게 처리하나요?**

- 의도: 상담 응답 프로세스와 권한 체계 파악
- 예: "센터 실장이 직접 확인" / "본사에서 1차 확인 후 각 센터 전달"

---

**IQ3. 게시 전에 승인이나 확인을 받아야 하는 콘텐츠가 있나요? 있다면 지금은 어떻게 처리하시나요?**

- 의도: 공식/비공식 검수 프로세스 파악
- 예: "카카오톡으로 확인받고 게시" / "별도 승인 없이 바로 수정"

---

**IQ4. 지금 관리자 페이지를 쓰면서 가장 자주 반복되는 비효율적인 업무가 있나요?**

- 의도: 자동화 또는 개선이 필요한 포인트 발굴
- 예: "센터별로 같은 공지를 복사해서 다시 등록" / "상담 신청자 엑셀 정리 후 수동 전달"

---

**IQ5. 여러 센터에 동시에 노출했던 콘텐츠 사례가 있다면 어떤 경우였나요?**

- 의도: 공통 콘텐츠 설계 기준 확보
- 예: "전체 센터 대상 오디션 안내" / "본사 정책 변경 공지"

---

**IQ6. 운영이 새 시스템으로 이관될 때 가장 걱정되는 점이 있다면 무엇인가요?**

- 의도: 실무 저항과 위험 요소 파악
- 예: "기존보다 사용이 어려워질까 걱정" / "센터별 권한이 애매해질까 걱정"

---

**IQ7. 이번 리뉴얼에서 꼭 반영되었으면 하는 점을 한 가지만 말씀해주세요.**

- 의도: 응답자의 최우선 요구를 한 줄로 정리

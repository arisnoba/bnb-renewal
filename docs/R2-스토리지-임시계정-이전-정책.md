# R2 스토리지 임시 계정 이전 정책

## 결론

클라이언트 Cloudflare R2 계정을 아직 받지 못했다면 개발자 계정으로 먼저 테스트해도 된다. 단, 운영 전환은 "버킷 소유권을 그대로 넘긴다"가 아니라 아래 둘 중 하나로 본다.

1. 프로젝트 전용 Cloudflare 계정을 만들고 클라이언트를 Super Administrator로 추가한 뒤 개발자가 빠진다.
2. 개발자 계정의 R2 버킷에서 클라이언트 계정의 R2 버킷으로 객체를 복사한다.

이번 프로젝트에서는 2번을 기본 전제로 둔다. 이미 여러 프로젝트가 들어 있는 개인 Cloudflare 계정 자체를 클라이언트에게 넘기면 안 된다.

## 설계 원칙

DB에는 Cloudflare 계정에 종속된 URL을 저장하지 않는다.

저장해야 하는 값:

- `objectKey`
- `bucket`
- `assetRole`
- `contentType`
- `size`
- `checksum` 또는 원본 식별자
- `legacy_meta.originalPath`
- `legacy_meta.sourceDb`
- `legacy_meta.sourceTable`
- `legacy_meta.sourceId`

저장하지 말아야 하는 값:

- `https://...r2.dev/...` 전체 URL
- 특정 Cloudflare `account_id`가 들어간 S3 endpoint
- 임시 테스트 버킷의 public URL

서비스 URL은 런타임 설정으로 조합한다.

```text
R2_PUBLIC_BASE_URL=https://media.example.com
objectKey=legacy/news/baewoo/new_notice/123/file-0/foo.jpg
publicUrl=${R2_PUBLIC_BASE_URL}/${objectKey}
```

이렇게 두면 개발자 R2에서 클라이언트 R2로 옮긴 뒤에도 DB 전체 치환이 아니라 환경변수와 배포 설정 변경으로 끝낼 수 있다.

## 개발자 계정으로 테스트할 때의 규칙

- 테스트 버킷과 운영 버킷 이름을 분리한다.
- `r2.dev` URL은 검증용으로만 사용한다.
- 앱 코드와 Payload 데이터에는 `objectKey` 중심으로 저장한다.
- 업로드 스크립트는 재실행 가능해야 한다.
- 같은 `objectKey`가 이미 있으면 기본적으로 skip 또는 checksum 비교 후 갱신한다.
- 최종 전환 전에는 클라이언트 계정에서 새 API token을 발급받아 환경변수만 교체한다.

권장 버킷명:

```text
bnb-renewal-dev
bnb-renewal-prod
```

권장 key 구조:

```text
legacy/{collection}/{source_db}/{bo_table}/{source_id}/{asset_role}/{filename}
media/{collection}/{slug-or-id}/{asset_role}.{ext}
```

`legacy/*`는 원본 추적용이고, `media/*`는 최종 서비스용이다.

## 이전 방법

### 방법 A: R2에서 R2로 전체 복사

Cloudflare R2는 S3 호환 API를 제공하므로 `rclone`, S3 호환 CLI, 또는 Cloudflare Super Slurper 계열 도구로 복사할 수 있다.

흐름:

```text
개발자 R2 bucket
  -> objectKey 유지 복사
  -> 클라이언트 R2 bucket
  -> R2_PUBLIC_BASE_URL 교체
  -> 샘플 URL/이미지 검증
```

3GB대 스토리지라면 전체 복사 자체는 부담이 큰 작업은 아니다. 중요한 것은 복사 난이도가 아니라 DB가 임시 URL에 묶이지 않도록 미리 막는 것이다.

### 방법 B: 프로젝트 전용 Cloudflare 계정 이전

처음부터 프로젝트 전용 Cloudflare 계정을 만들 수 있다면 가장 깔끔하다.

흐름:

```text
개발자 이메일로 프로젝트 전용 Cloudflare 계정 생성
  -> R2 테스트/운영 버킷 생성
  -> 클라이언트 이메일을 Super Administrator로 초대
  -> 결제/소유 확인
  -> 개발자 계정 제거
```

개인 Cloudflare 계정에 다른 프로젝트가 섞여 있다면 이 방법은 사용하지 않는다.

## 환경변수 기준

앱과 스크립트는 아래 값만 바꾸면 이전 가능해야 한다.

```text
R2_ACCOUNT_ID=
R2_ACCESS_KEY_ID=
R2_SECRET_ACCESS_KEY=
R2_BUCKET=
R2_PUBLIC_BASE_URL=
R2_ENDPOINT=
```

Payload/PostgreSQL에 저장되는 값은 `R2_PUBLIC_BASE_URL`을 포함하지 않는 `objectKey`를 기본으로 한다.

## 최종 운영 전 체크리스트

- 클라이언트 Cloudflare 계정에서 R2 사용 가능 여부 확인
- 클라이언트 계정에 운영 버킷 생성
- 개발자 계정 테스트 버킷의 객체를 클라이언트 운영 버킷으로 복사
- 객체 수, 총 용량, 샘플 checksum 또는 파일 크기 비교
- `R2_PUBLIC_BASE_URL`을 운영 도메인으로 변경
- Payload Media 또는 콘텐츠 레코드의 `objectKey`가 그대로 동작하는지 확인
- 개발자 계정 API token 폐기

## 현재 결정

- 지금은 개발자 계정 R2로 테스트 가능하다.
- DB에는 URL이 아니라 `objectKey`를 저장한다.
- 운영 전에는 클라이언트 R2로 복사하고 환경변수만 교체한다.
- `public/legacy`는 계속 로컬 staging/cache로 두고, R2는 그 다음 단계의 원격 저장소로 본다.

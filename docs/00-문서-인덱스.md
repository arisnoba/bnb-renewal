# 문서 인덱스

> 목적: 이 저장소의 문서 진입점을 줄이고, 반복되는 정책을 하나의 기준 문서에서 관리한다.
> 마지막 갱신: 2026-07-14

## 운영 원칙

- 아래 "활성 문서"를 현재 기준으로 사용한다.
- `docs/archive`와 `plan/archive`는 과거 판단 근거와 상세 기록이다.
- 활성 문서와 archive 문서가 충돌하면 활성 문서를 우선한다.
- 새 정책이 생기면 새 파일을 만들기보다 가장 가까운 활성 문서를 갱신한다.
- 작업 진행 기록은 `plan/현재작업.md`에 남긴다.
- DB 관련 내용을 문서에 남길 때는 `로컬 MariaDB`, `로컬 Postgres/Payload DB`, `원격 Vercel Neon DB`처럼 대상 DB를 구분해서 적는다.

# 배우앤배움 통합 사이트 GNB 설계 대안 비교

## 1. 핵심 전제

백엔드, DB, 콘텐츠 관리 구조는 **두 안 모두 동일하게 통합 운영**합니다.

이 문서가 다루는 것은 단 하나입니다.

> **"우리 사이트를 방문하는 사용자가 누구인가"에 따른 GNB(상단 네비게이션) 설계 선택**

---

## 2. 두 가지 사용자 유형

배우앤배움 사이트에 방문하는 사용자는 크게 두 유형으로 나뉩니다.

| 유형 | 특징 | 예시 |
|------|------|------|
| **목적형** | 센터를 모르거나, 배우고 싶은 것/목적이 먼저 | "연기 배우고 싶은데 프로그램 뭐가 있지?" "아이 연기 교육 어디서 하지?" |
| **센터 인지형** | 특정 센터를 이미 알고 직접 찾아오는 | "입시센터 등록하러 왔어" "아이 키즈센터 다니는데 강사진 보려고" |

**GNB 설계의 핵심 질문:**

> 우리 사이트 방문자 중 어떤 유형이 더 많은가?

---

## 3. 대안 1. 콘텐츠 카테고리 중심 GNB

### GNB 구조

```
게이트 | 배우앤배움 소개 | 센터 소개 | 프로그램 | 강사진 | 성과/활동 | 캐스팅/오디션 | 이용안내 | 상담/지원
```

### 목적형 사용자 탐색 시나리오

```
"연기 프로그램 보고 싶어"
→ GNB [프로그램] 클릭
→ 전체 프로그램 목록 (센터 필터 선택 가능)
→ 원하는 프로그램 상세 → 상담 신청
```

```
"강사가 누가 있는지 보고 싶어"
→ GNB [강사진] 클릭
→ 전체 강사진 목록 (센터/분야 필터)
→ 강사 상세 → 상담 신청
```

### 센터 인지형 사용자 탐색 시나리오

```
"입시센터 찾아왔어"
→ GNB [센터 소개] → 드롭다운에서 입시센터 선택
→ 입시센터 랜딩
→ 입시센터 내 프로그램/강사진/성과 탐색
```

### 특징

- 센터는 "센터 소개" 메뉴 하위와 각 콘텐츠의 필터로 접근
- 처음 방문자가 목적 기반으로 바로 탐색 가능
- 배우앤배움 브랜드가 전면에 노출

---

## 4. 대안 2. 센터 브랜드 중심 GNB

### GNB 구조

```
배우앤배움 | 아트센터 | 입시센터 | 키즈센터 | 하이틴센터 | 애비뉴센터
[오디션] [상담하기] — 헤더 우측 CTA 버튼
```

> `오디션`과 `상담하기`는 GNB 항목이 아니라 헤더 CTA 버튼으로 배치.
> GNB에 센터 엔티티(브랜드)와 기능 유형을 섞으면 범주 불일치 발생.

### 각 센터 공통 2차 메뉴

```
센터소개 | 프로그램 | 강사진 | 성과/활동 | 캐스팅/오디션 | 이용안내 | 상담
```

### 센터 인지형 사용자 탐색 시나리오

```
"입시센터 찾아왔어"
→ GNB [입시센터] 클릭 (바로 진입)
→ 입시센터 내 프로그램/강사진/성과 탐색
→ 상담 신청
```

### 목적형 사용자 탐색 시나리오

```
"연기 프로그램 보고 싶어"
→ GNB에서 어느 센터를 눌러야 할지 모름
→ 배우앤배움 메뉴 탐색 or 센터 하나씩 클릭해봄
→ 탐색 경로가 늘어남
```

> 목적형 사용자는 센터를 먼저 골라야 하는 선택 장벽을 만나게 됩니다.

### 특징

- 기존 수강생의 재방문 경험 그대로 유지
- 센터 브랜드가 1차 메뉴에서 바로 노출
- 각 센터 클릭 시 해당 센터 필터가 선택된 상태로 콘텐츠 진입

---

## 5. GNB 비교 요약

| 항목 | 콘텐츠 카테고리 중심 | 센터 브랜드 중심 |
|------|:---:|:---:|
| **유리한 사용자** | 목적형 (처음 방문) | 센터 인지형 (알고 온) |
| **첫 화면 노출** | 콘텐츠 카테고리 | 센터 브랜드 |
| **센터 진입 경로** | 센터 소개 메뉴 or 필터 | GNB 1차 클릭 |
| **목적 기반 탐색** | 바로 가능 | 센터 선택 후 가능 |
| **배우앤배움 브랜드** | 전면 노출 | 센터 뒤에 위치 |
| **모바일 GNB 항목 수** | 9개 | 6개 (CTA 별도) |

> 내부 구조, DB, 필터 동작은 두 안 모두 동일합니다.

| 센터 간 통합 프로모션/패키지를 기획할 가능성이 있는가? | ✓ | |
| 배우앤배움 브랜드를 전면에 내세우고 싶은가? | ✓ | |

## 7. 추가 의견

- `오디션`은 전환성이 높으므로 어느 안을 선택해도 헤더 CTA 또는 눈에 잘 띄는 위치에 배치를 권장합니다.
- `상담하기`보다 `무료상담` 또는 `상담 신청` 문구가 전환에 유리합니다.
- 애비뉴센터는 현재 단일 페이지 구조로, 어느 안이든 콘텐츠 보강이 선행되어야 통합 편입이 가능합니다.

---

## 8. 결론

> **두 안의 차이는 GNB 하나뿐입니다.**
> 백엔드, DB, 콘텐츠 구조는 동일하게 가져갑니다.

선택 기준은 하나입니다.

**"우리 사이트를 찾아오는 사람의 대부분은 목적이 먼저인가, 센터가 먼저인가?"**

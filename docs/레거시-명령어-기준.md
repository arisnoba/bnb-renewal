# 레거시 마이그레이션 명령어 기준

## 목적

`c0`, `p0`, `p1` 같은 내부 단계명은 현재 작업자가 보기 어렵다. 앞으로 실행 명령은 실제 대상과 역할이 드러나는 이름을 쓴다.

## 명령어 그룹

### `legacy:db:*`

로컬 MariaDB 컨테이너와 원본 dump 복원에 사용한다.

```bash
npm run legacy:db:up
npm run legacy:db:import
npm run legacy:db:verify
```

기존 로컬 MariaDB를 버리고 다시 복원할 때만 사용한다.

```bash
npm run legacy:db:import:reset
```

### `legacy:work:*`

센터별 MariaDB 원본 테이블을 `bnb_legacy_work`의 work table로 합치는 명령이다.

예시:

```bash
npm run legacy:work:news
npm run legacy:work:teachers
npm run legacy:work:profiles
npm run legacy:work:casting-appearances
```

이 단계는 PostgreSQL/Payload에 넣는 작업이 아니다. MariaDB 안에서 검증하기 좋은 중간 테이블을 만드는 단계다.

### `legacy:assets:*`

MariaDB work table을 기준으로 FTP 이미지 존재 확인과 다운로드를 수행한다.

Dry-run:

```bash
npm run legacy:assets:news:dry-run -- --sample-size 100 --progress-every 10
npm run legacy:assets:profiles:dry-run
npm run legacy:assets:teachers:dry-run
```

Download:

```bash
npm run legacy:assets:news:download -- --progress-every 100
npm run legacy:assets:profiles:download
npm run legacy:assets:teachers:download
```

리포트는 `tmp/legacy-assets/`에 생성된다. 다운로드 파일은 `public/legacy/` 아래에 생성된다.

### `payload:legacy:*`

로컬 Postgres/Payload로 옮기기 전후의 백업, 스냅샷, 이미지 경로 치환 같은 Payload 적재 보조 명령이다.

```bash
npm run payload:legacy:snapshot
npm run payload:legacy:backup
npm run payload:legacy:scan-urls
npm run payload:legacy:upload-images
npm run payload:legacy:replace-image-paths
```

기본적으로 `config/env/local-postgres.env`를 사용하므로 로컬 Postgres를 대상으로 한다.

### `db:*`, `payload:*`, `dev`

현재 기본값은 로컬 Postgres/Payload 기준이다.

```bash
npm run db:local:up
npm run db:migrate
npm run dev
npm run payload
```

원격 Neon을 직접 봐야 할 때만 `remote` 접미사를 쓴다.

```bash
npm run db:remote:migrate
npm run dev:remote
npm run payload:remote
```

## MariaDB 테스트 페이지를 브라우저에서 보는 방법

관리자 `/admin`은 Payload/Postgres를 본다. 반면 `/test/mariadb/*`는 로컬 MariaDB의 `bnb_legacy_work` 테이블을 본다.

따라서 MariaDB 테스트 페이지를 보려면 아래 순서로 실행한다.

```bash
npm run legacy:db:up
npm run legacy:work:news
npm run legacy:work:teachers
npm run legacy:work:profiles
npm run dev
```

브라우저:

```text
http://localhost:3000/test/mariadb
http://localhost:3000/test/mariadb/news
http://localhost:3000/test/mariadb/teachers
http://localhost:3000/test/mariadb/profiles
```

특정 테이블이 비어 있거나 컬럼 오류가 나면 해당 `legacy:work:*` 명령을 먼저 다시 실행한다.

## 현재 사용하지 않는 이름

앞으로 새 명령이나 문서에는 아래 이름을 쓰지 않는다.

- `db:c0:*`
- `db:seed:c0-*`
- `db:seed:p0-*`
- `db:seed:p1-*`

과거 계획서나 archive 문서의 `c0` 표기는 당시 작업 단계 기록으로만 본다.

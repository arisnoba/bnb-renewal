# 배우앤배움 통합 사이트 메뉴 구조도

## 1. 개편 전제

- 최상위 브랜드는 `배우앤배움`
- 하위 브랜드는 `아트센터 / 입시센터 / 키즈센터 / 하이틴센터 / 애비뉴센터`
- 첫 진입은 `게이트 페이지`로 받고, 사용자는 바로 원하는 센터로 이동 가능
- 기존처럼 센터마다 동일 메뉴를 반복하지 않고, 공통 메뉴는 통합
- 공통 메뉴 내부에서 `센터 / 대상 / 목적 / 프로그램 유형` 기준으로 필터링

## 3. GNB 권장안

상단 GNB는 너무 많아지지 않게 아래 8개 축으로 정리하는 것이 적절합니다.

1. 게이트
2. 배우앤배움 소개
3. 센터 소개
4. 프로그램
5. 강사진
6. 성과/활동
7. 이용안내
8. 상담/지원

`캐스팅/오디션`은 중요도가 높기 때문에 `성과/활동`과 별도 GNB로 빼도 되며, 운영 우선순위에 따라 아래 두 방식 중 선택 가능합니다.

- 확장형 GNB: `게이트 / 소개 / 센터 / 프로그램 / 강사진 / 성과·활동 / 캐스팅·오디션 / 이용안내 / 상담`
- 축약형 GNB: `게이트 / 소개 / 센터 / 프로그램 / 강사진 / 성과·활동 / 이용안내 / 상담`

축약형을 쓸 경우 `캐스팅/오디션`은 `성과·활동` 내부의 강한 CTA 섹션으로 노출합니다.

## 4. 센터별 진입 구조

각 센터는 별도 서브홈을 가지되, 메뉴 체계는 공통 유지가 좋습니다.

```text
/centers/art
/centers/exam
/centers/kids
/centers/highteen
/centers/avenue
```

각 센터 서브홈 공통 구성:

- 센터 소개
- 추천 대상
- 대표 프로그램
- 강사진
- 성과/포트폴리오
- 상담 CTA

즉, `센터별 사이트`가 아니라 `통합 사이트 안의 센터 랜딩` 개념으로 운영합니다.

## 5. 통합 필터 구조

공통 메뉴를 통합하려면 각 목록 페이지에서 아래 필터가 기본으로 들어가야 합니다.

### 프로그램 필터

- 센터: 아트 / 입시 / 키즈 / 하이틴 / 애비뉴
- 대상: 아동 / 청소년 / 입시생 / 성인 / 40세 이상
- 목적: 취미 / 데뷔 / 입시 / 캐스팅 / 매니지먼트
- 유형: 정규과정 / 특강 / 위탁교육 / 프로필촬영 / 오디션대비

### 강사진 필터

- 센터
- 분야
- 대상 연령
- 담당 프로그램

### 성과/활동 필터

- 센터
- 유형: 합격 / 출연 / 후기 / 영상 / 뉴스 / 공지
- 기간
- 키워드

### 캐스팅/오디션 필터

- 센터
- 유형: 드라마 / 광고 / 오디션 / 섭외
- 상태: 모집중 / 진행중 / 종료
- 대상 연령

## 6. 센터별 고유 콘텐츠 배치 원칙

공통 메뉴로 묶되, 각 센터 고유 자산은 아래처럼 흡수하는 방식이 적절합니다.

### 아트센터

- `프로그램` 내: 등급제, 위탁교육, 프로필 촬영
- `캐스팅/오디션` 내: 드라마/광고 캐스팅, 오디션 지원
- `성과/활동` 내: 출연장면, BNB출신 아티스트, 촬영후기

### 입시센터

- `프로그램` 내: 입시반, 재입시반, 편입반, 예비입시반, 예고입시반
- `성과/활동` 내: 대학 합격현황, 예고 합격현황, 합격후기, 합격영상
- `이용안내` 내: 장학제도, 특별한 시스템

### 키즈센터

- `프로그램` 내: 영재 교육과정, 아역배우 과정, 아티스트 과정
- `캐스팅/오디션` 내: 아역 대상 드라마/광고 캐스팅
- `성과/활동` 내: 아역배우 프로필, 출연현황

### 하이틴센터

- `프로그램` 내: 하이틴 특강, 이달의 커리큘럼
- `캐스팅/오디션` 내: 진행중 캐스팅, 다이렉트 캐스팅, 오디션 지원
- `성과/활동` 내: 섭외뉴스, 출연장면, BNB 루키

### 애비뉴센터

- `프로그램` 내: 40세 이상 전용 과정
- `강사진` 내: 애비뉴 전용 강사진
- `이용안내` 내: 수업 시간, 운영 방식, 상담 동선

## 8. 최종 권장 메뉴 구조

실제 사용자 기준으로는 아래 구조가 가장 명확합니다.

```text
GNB
├─ 게이트
├─ 배우앤배움 소개
├─ 센터 소개
├─ 프로그램
├─ 강사진
├─ 성과/활동
├─ 캐스팅/오디션
├─ 이용안내
└─ 상담/지원
```

핵심 원칙은 다음 3가지입니다.

- 센터를 메뉴로 반복하지 않고 `센터 소개`와 `필터`로 푼다
- 공통 콘텐츠는 하나의 카테고리로 통합하고 센터별로 분기한다
- 게이트 페이지는 브랜드 허브이자 빠른 진입 페이지로만 쓰고, 실제 SEO/전환 콘텐츠는 통합 본 사이트 안에서 운영한다

## 9. 한 줄 결론

이번 통합 구조는 `5개 사이트를 그대로 병렬 운영하는 방식`보다, `배우앤배움 통합 사이트 + 게이트 페이지 + 센터별 필터 랜딩` 구조가 가장 효율적입니다.

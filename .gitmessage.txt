# [Conventional Commits 규격]
#   feat     : 새로운 기능 추가
#   fix      : 버그 수정
#   docs     : 문서 수정
#   style    : 코드 포맷팅, 세미콜론 누락, 코드 변경 없는 경우
#   refactor : 코드 리팩토링
#   test     : 테스트 코드 추가/수정
#   chore    : 빌드 업무 수정, 패키지 매니저 수정 등
#
# <type>: <description> (예: feat: 회원가입 API 연동)
# - 반드시 한국어로 작성하세요.
# - 본문이 필요한 경우 헤더에서 한 줄 띄우고 아래에 한국어로 작성하세요.
